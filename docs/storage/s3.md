# S3 Cloud Storage

## Overview

GhostCTL provides unified S3-compatible storage management supporting multiple cloud providers.

## Supported Providers

| Provider | Endpoint Configuration |
|----------|----------------------|
| Amazon S3 | Region-based (us-east-1, etc.) |
| MinIO | Custom endpoint URL |
| Azure Blob | `https://<account>.blob.core.windows.net` |
| Backblaze B2 | `https://s3.<region>.backblazeb2.com` |
| Wasabi | `https://s3.<region>.wasabisys.com` |
| DigitalOcean | `https://<region>.digitaloceanspaces.com` |
| Custom | Any S3-compatible endpoint |

## Configuration

### Configure Provider
```bash
ghostctl storage menu
# Select: S3/MinIO Storage Management > Configure S3 Provider
```

Configuration steps:
1. Select provider type
2. Enter endpoint/region
3. Provide access key and secret key
4. Set default bucket (optional)
5. Name the profile

### Profile Management
```bash
# Through menu:
ghostctl storage menu
# Select: S3/MinIO Storage Management > Manage Profiles
```

Profile operations:
- List Profiles
- Switch Profile
- Delete Profile
- Export Profile (shows credentials for CLI)


```bash
# Menu: Bucket Operations > Create Bucket
```

### Delete Bucket
```bash
# Menu: Bucket Operations > Delete Bucket
# Option to force delete (removes all objects first)
```

### Bucket Configuration

**Versioning:**
- Enable/suspend object versioning
- Preserves previous versions of objects

**Lifecycle Rules:**
- Delete old versions after N days
- Move to Glacier after N days
- Abort incomplete multipart uploads
- Custom lifecycle JSON

**Bucket Policy:**
- Public Read
- Private (default)
- Public Read-Write (dangerous)
- Custom JSON policy

## Restic Integration

GhostCTL integrates S3 storage with Restic for encrypted backups.

### Configure Restic S3 Backend
```bash
# Menu: Restic Integration > Configure Restic S3 Backend
```

Sets up:
- Restic repository URL
- Repository password
- AWS credentials
- Environment file at `~/.config/ghostctl/restic-s3.env`

### Initialize Repository
```bash
# Menu: Restic Integration > Initialize Restic Repository
```

### Backup Operations
- Create Backup to S3
- Restore from S3
- List Snapshots
- Prune Old Snapshots

### Prune Policy
Configure retention:
- Keep last N snapshots
- Keep daily for N days
- Keep weekly for N weeks
- Keep monthly for N months

## Environment Variables

For CLI usage outside GhostCTL:

```bash
export AWS_ACCESS_KEY_ID="your-key"
export AWS_SECRET_ACCESS_KEY="your-secret"
export AWS_DEFAULT_REGION="us-east-1"
export AWS_ENDPOINT_URL="https://your-endpoint"  # For non-AWS
```

## MinIO Setup

For self-hosted MinIO:

1. Configure with custom endpoint: `http://minio.local:9000`
2. Region defaults to `us-east-1`
3. Use MinIO access/secret keys

## Best Practices

- Use separate profiles for different environments
- Enable versioning for important buckets
- Set lifecycle rules to manage storage costs
- Test backups with regular restore verification
- Use presigned URLs for temporary access sharing
